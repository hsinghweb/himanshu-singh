Trino_004: Running Trino

Learn how to run Trino using Docker and understand its key concepts.

üê≥ Running with Docker

Start Trino using Docker with a single command:

docker run -d -p 8080:8080 --name trino-trial trinodb/trino

This command:

  • Downloads the container image
  • Names it trino-trial
  • Runs it in the background
  • Maps port 8080 to your workstation

üíª Using the CLI

Connect to the container and run queries:

$ docker exec -it trino-trial trino
trino> select count(*) from tpch.sf1.nation;
_col0------
25
(1 row)

üîë Key Concepts

Understanding these concepts is crucial for working with Trino effectively.

Connector

  • Adapts Trino to a data source
  • Every catalog is associated with a specific connector

Catalog

  • Defines details for accessing a data source
  • Contains schemas
  • Configures a specific connector to use

Schema

  • A way to organize tables
  • Combined with catalog defines a set of queryable tables

Table

  • A set of unordered rows
  • Organized into named columns with data types

üìù Additional Notes

  • Use USE catalog.schema to set default context
  • Start CLI with --catalog and --schema for specific context
  • Access via JDBC driver available for Java applications
  • Type quit, exit, or press Ctrl-D to exit CLI
