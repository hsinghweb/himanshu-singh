Trino_002: Technical Terms Explained

Let's dive deeper into each of the key technical terms from the Trino explanation.

üîπ 1. Interactive Querying

What it means:

"Interactive querying" refers to the ability to run SQL queries and get results quickly, often within seconds, so that humans can interact with the system in real-time.

Context in Trino:

  • Traditional big data systems (like Hadoop) can be slow ‚Äî taking minutes or hours to return results.
  • Trino is designed for low-latency execution, so analysts, engineers, or data scientists can:
    • Run ad hoc queries
    • Explore data on the fly
    • Iterate fast in tools like Jupyter, Superset, or SQL editors

Example:

Instead of waiting 20 minutes for a Hive job to finish, Trino can return the same result in under 10 seconds.

üîπ 2. Massively Parallel Processing (MPP)

What it means:

MPP is an architecture where a query is broken down into smaller pieces and executed in parallel across many nodes or servers.

Context in Trino:

  • Trino uses a coordinator + worker nodes model:
    • The coordinator breaks a SQL query into smaller tasks (called "fragments")
    • The worker nodes execute those tasks in parallel, handling:
      • Table scans
      • Joins
      • Aggregations
      • Filtering, etc.
  • This parallelism leads to high performance, especially on large datasets.

Example:

If you're querying 10 TB of data, Trino splits the work across 100 worker nodes ‚Äî each node processes 100 GB simultaneously. This speeds up the query significantly.

üîπ 3. Query Federation Engine

What it means:

A query federation engine lets you query data from multiple, heterogeneous sources ‚Äî as if it were a single database.

Context in Trino:

  • Trino supports connectors to many systems: Hive, MySQL, S3, MongoDB, BigQuery, etc.
  • You can join a MySQL table with a Parquet file in S3 in a single SQL query.
  • This eliminates the need to ETL (extract, transform, load) data into a central warehouse.

Example SQL in Trino:

SELECT s.name, o.total
FROM mysql.sales.customers s
JOIN hive.analytics.orders o
ON s.customer_id = o.customer_id
WHERE o.total > 1000;

This query:

  • Reads from MySQL (`mysql.sales.customers`)
  • Joins with a table in S3 or HDFS (`hive.analytics.orders`)
  • Runs without moving the data beforehand
